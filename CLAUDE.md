# 项目开发规范

## 项目概述
基于大模型的中小学作业批改系统。学生上传纸质作业照片，系统自动完成切题、OCR识别、大模型批改，并在原图上生成批注和评分。

## 技术栈
- 后端：Python + Flask
- 前端：HTML + CSS + 原生JavaScript（Flask模板）
- 图像处理：OpenCV + 阿里云/百度云图像增强API
- 切题：阿里云 RecognizeEduPaperCut
- OCR：阿里云 RecognizeEduQuestionOcr
- 大模型：通义千问 / DeepSeek API

## AI 编码规则

### 代码风格
- Python 代码遵循 PEP 8 规范
- 使用 type hints（Python 3.10+ 语法）
- 函数和类必须有简要中文注释说明用途
- 变量命名使用 snake_case，类名使用 PascalCase
- 配置项（API Key、端点URL等）必须从环境变量或配置文件读取，禁止硬编码

### 代码组织
- 后端模块按功能划分：`preprocessing/`、`ocr/`、`grading/`、`annotation/`、`api/`
- 每个模块对外暴露清晰的接口函数
- 避免循环依赖，模块间通过接口函数调用

### 错误处理
- 所有外部API调用必须有超时设置和异常捕获
- 大模型API调用失败需实现重试机制（最多3次）
- OCR置信度低于阈值时标记为"需人工复核"
- 图像质量不达标时返回明确的错误提示

### 安全规范
- API Key 等敏感信息存放在 `.env` 文件中，`.env` 已在 `.gitignore` 中
- 文件上传需校验类型和大小，防止恶意文件
- 返回的图像数据需做安全处理

### 测试要求
- 每个核心模块编写对应的单元测试
- 测试文件放在 `tests/` 目录下，命名为 `test_<模块名>.py`

### 文档规范
- 需求文档：`docs/需求文档.md`
- 进度表：`docs/进度表.md`

### Git 提交规范
- commit message 使用中文
- 遵循约定式提交格式：`<类型>(<范围>): <描述>`
- 常用类型：
  - `feat`：新功能
  - `fix`：修复 bug
  - `docs`：文档变更
  - `style`：代码格式调整（不影响逻辑）
  - `refactor`：重构（既非新功能也非修复）
  - `test`：添加或修改测试
  - `chore`：构建过程或辅助工具变更
- 示例：`feat(ocr): 对接阿里云 OCR 识别接口`、`fix(api): 修复文件上传超时问题`
- 禁止提交敏感信息（.env、API Key 等）
- 提交前确认 `.gitignore` 已覆盖不应提交的文件类型

## 进度表自动更新规则

**每次完成功能任务后，必须同步更新 `docs/进度表.md`：**

1. **更新模块状态行**：将对应任务的"状态"从"未开始"改为"进行中"或"已完成"，并填写"最近更新"为当天日期
2. **更新总体进度**：重新计算所属阶段的完成百分比，更新"总体进度"表
3. **追加更新日志**：在底部"更新日志"表格中追加一行，简述本次完成的内容
4. **更新规则**：如果新增了任务项（如需求变更），在对应阶段的分模块表中追加行

状态定义：
- `未开始`：尚未着手
- `进行中`：代码已开始编写但未完全完成
- `已完成`：代码完成且基本功能可用
- `待验证`：代码完成，需要集成测试验证

# 运行测试
python -m pytest tests/
