# 脚本说明

本文档说明当前仓库中主要脚本的用途、输入输出和适用场景，便于后续继续实验与整理流程。

## 一、图像预处理类

### 1. `paper_perspective_correction.py`
- 用途：对拍摄试卷做透视矫正，输出近似俯视视角图像。
- 适用场景：原始拍照图存在倾斜、边缘透视畸变时。
- 结果作用：为后续增强、切题、OCR 提供更规整输入。

### 2. `validate_perspective_correction.py`
- 用途：验证透视矫正流程的结果和中间步骤。
- 适用场景：调试透视矫正效果时。

### 3. `document_enhance.py`
- 用途：把纠正后的试卷图增强成白底黑字图。
- 输入：透视矫正后的图像。
- 输出：增强图，通常保存到 `tests/paper_samples/outputs/enhanced_document/`。
- 结果作用：为阿里云、Kimi、SeedDream 等后续步骤提供更稳定输入。

## 二、切题主流程类

### 4. `adaptive_paper_cut.py`
- 用途：自适应切题主流程入口。
- 核心逻辑：
  1. 先用 Kimi 判断页面更偏手写还是印刷。
  2. 若偏手写，则走 `seeddream_qieti.py`。
  3. 若偏印刷，则走 `aliyun_paper_cut.py`。
- 输出：保存到 `tests/paper_samples/outputs/adaptive_results/`。
- 适用场景：当前阶段的混合切题实验主流程。

### 5. `seeddream_qieti.py`
- 用途：调用 SeedDream 在原图上直接画红框。
- 核心逻辑：
  1. 可选先用 Kimi 对页面题目做摘要分析。
  2. 再把图像和提示词发送给 SeedDream。
  3. 返回一张已经画好红框的图。
- 适用场景：整页主要是手写题目、手写作答时。
- 局限：本质是图像编辑，不是严格检测模型。

### 6. `aliyun_paper_cut.py`
- 用途：调用阿里云 `RecognizeEduPaperCut` 做题目区域切分。
- 支持：
  - `cut_type=question`
  - `cut_type=answer`
- 输出：
  - 原始 JSON
  - overlay 图
- 适用场景：整页偏规则印刷题、排版明确时。

### 7. `aliyun_paper_structed.py`
- 用途：调用阿里云 `RecognizeEduPaperStructed` 做更细粒度的结构化版面分析。
- 输出：
  - 原始结构化 JSON
  - 结构化 overlay 图
- 当前 overlay 图层说明：
  - `layout:*`：版面块
  - `subject:*`：题目级主块
  - `element:*`：题目内部元素块
  - `content`：更细粒度内容块
- 适用场景：分析页面结构、研究更细粒度后处理规则时。

## 三、切题后处理类

### 8. `redbox_crop.py`
- 用途：从 SeedDream 已画好红框的图中检测红框并裁出每个框内区域。
- 核心逻辑：
  1. HSV 提取红色像素。
  2. 轮廓检测红框。
  3. 按从上到下、从左到右排序。
  4. 批量裁出 `q_01.png`、`q_02.png` 等。
- 输出：
  - 裁图目录
  - `manifest.json`
  - 检测框 overlay
- 适用场景：手写页经过 SeedDream 画框后，进一步得到单题图像。

### 9. `recursive_paper_cut.py`
- 用途：递归式切题实验脚本。
- 核心逻辑：对切出的区域继续调用阿里云切题，再尝试把大块拆成小块。
- 适用场景：研究“切后再切”的递归策略。
- 当前状态：实验性强，主要用于分析，不是当前主流程入口。

## 四、OCR 与识别类

### 10. `aliyun_question_ocr.py`
- 用途：调用阿里云 `RecognizeEduQuestionOcr` 对单题图像做题目 OCR。
- 输入：切分后的题目小图，例如 `q_01.png`。
- 输出：原始 JSON，可包含 `content`、`prism_wordsInfo`、字符级信息等。
- 适用场景：切完题后研究单题 OCR 效果。

### 11. `kimi_question_grading.py`
- 用途：调用 Kimi 对单题图进行题目识别和判对错。
- 输出字段：
  - `question_text`
  - `student_answer`
  - `correct_answer`
  - `is_correct`
  - `explanation`
  - `mistake_analysis`
  - `knowledge_points`
  - `confidence`
- 适用场景：切完题后，直接对单题做理解和批改。

### 12. `llm_paper_cut.py`
- 用途：使用视觉大模型直接对整页做题目识别、题框估计和批改。
- 说明：这是早期整页 LLM 路线实验脚本。
- 当前作用：保留作参考，对比与实验，不是当前主流程。

## 五、第三方或旧路线实验类

### 13. `youdao_paper_cut.py`
- 用途：调用有道切题接口。
- 当前作用：保留作对比实验，不是当前主流程。

### 14. `test.py`
- 用途：临时测试脚本。
- 当前作用：用于快速验证某条 OCR 或识别链路，不属于稳定流程脚本。

## 六、当前建议主链

当前较稳定的实验主链为：

1. `paper_perspective_correction.py`：透视矫正
2. `document_enhance.py`：增强成白底黑字图
3. `adaptive_paper_cut.py`：Kimi 路由选择切题方式
4. 若走手写路线：
   - `seeddream_qieti.py`
   - `redbox_crop.py`
5. 若走印刷路线：
   - `aliyun_paper_cut.py`
6. 单题识别或判对错：
   - `aliyun_question_ocr.py`
   - 或 `kimi_question_grading.py`

## 七、输出目录说明

### `tests/paper_samples/outputs/enhanced_document/`
增强后的白底黑字输入图。

### `tests/paper_samples/outputs/adaptive_results/`
自适应分流切题结果。

### `tests/paper_samples/outputs/redbox_crops_adaptive/`
从 SeedDream 红框图中裁出的单题图。

### `tests/paper_samples/outputs/aliyun_results/`
阿里云普通切题与结构化切题实验结果。

### `tests/paper_samples/outputs/kimi_question_results/`
Kimi 单题识别与批改输出。
